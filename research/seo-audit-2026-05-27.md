## Executive summary

1. **getcorder.com is not in Google's index at all.** `site:getcorder.com` returns zero results. Brand query "getcorder" doesn't surface us either. The most likely root cause is the next bullet, layered on top of being a new domain.
2. **Every subpage has `<link rel="canonical" href="https://getcorder.com/">`** pointing back to the root, including `/privacy-policy/`, `/refunds/`, `/terms/`, `/install/`, `/contact/`, `/thanks/`, `/404/`. This tells Google "ignore these, they are duplicates of the homepage." It is the highest-impact bug on the site. Caused by `metadata.alternates.canonical: "/"` in `src/app/layout.tsx` being inherited by every page that doesn't override it. Fix is a one-line `alternates.canonical` per page.
3. **Every subpage title is doubled:** "Install Corder | Corder | Corder", "Privacy Policy | Corder | Corder", because each `page.tsx` sets `title: "X | Corder"` and the root layout template wraps with `"%s | Corder"`. Two fixes possible, both small.
4. **JSON-LD `SoftwareApplication.offers` ships stale pricing** ($12 monthly, $99 annual) and stale `softwareVersion: "0.9"`. Real shipped lineup is Free $0, Pro $10 launch / $14 regular / $99 annual, Max $24 launch / $29 regular / $239 annual. Real release is 0.10.0. This contradicts on-page pricing and could trigger Google Merchant / rich-result mismatch flags.
5. **Ship-this-week list (in order):** (a) fix per-page canonical, (b) fix duplicated title template, (c) update JSON-LD pricing + version, (d) add `FAQPage` schema (10 Q&A already written), (e) request indexing in Search Console for the home and 4 sitemap URLs. Estimated effort 2-3 hours.

## 1. Crawlability and indexing

### robots.txt — fine
```
User-Agent: *
Allow: /

Host: https://getcorder.com
Sitemap: https://getcorder.com/sitemap.xml
```
Open to all, sitemap referenced. Nothing to fix.

### sitemap.xml — undersized
Currently lists 4 URLs: `/`, `/privacy-policy/`, `/refunds/`, `/terms/`.
Static export emits these additional indexable routes that are NOT in the sitemap:
- `/install/` — link from every Download CTA, legitimate destination
- `/contact/` — link from footer
- `/contact/sales/` — separate sales contact page
- `/thanks/` — post-checkout confirmation, has `index, follow`; should arguably be `noindex` since it's a state page
- `/404/` — not a destination, exclude from sitemap

`/signup/`, `/login/`, `/verify/`, `/account/` are user-state pages and should be `noindex, nofollow` (account-area, no SEO value).

Decision matrix:

| Route | Sitemap? | robots tag |
|---|---|---|
| `/` | yes | `index, follow` |
| `/install/` | yes | `index, follow` (it's a real instruction page) |
| `/contact/` | yes | `index, follow` |
| `/contact/sales/` | maybe | `index, follow` |
| `/privacy-policy/` | yes | `index, follow` |
| `/refunds/` | yes | `index, follow` |
| `/terms/` | yes | `index, follow` |
| `/thanks/` | no | `noindex, follow` |
| `/signup/`, `/login/`, `/verify/`, `/account/` | no | `noindex, nofollow` |
| `/404/` | no | `noindex, follow` (Next handles) |

### canonical — BROKEN

Every static export ships `<link rel="canonical" href="https://getcorder.com/"/>` regardless of route. Verified on `/install/`, `/contact/`, `/privacy-policy/`, `/refunds/`, `/terms/`, `/thanks/`, `/404/`.

Source: `src/app/layout.tsx` sets `metadata.alternates.canonical = "/"`. When a child page declares `metadata` without re-declaring `alternates`, Next.js inherits the parent's relative `"/"`, which `metadataBase` resolves to `https://getcorder.com/`.

Effect: Google's canonicalisation algorithm sees every subpage say "the real one is `/`", and refuses to index `/privacy-policy/`, `/refunds/`, `/terms/`, `/install/`, `/contact/`. They will eventually be marked "Duplicate, Google chose different canonical than user" in Search Console.

**Fix recipe:**

In every child `page.tsx` (`install`, `contact`, `contact/sales`, `privacy-policy`, `refunds`, `terms`, `thanks`, `signup`, `login`, `verify`, `account`) add:
```ts
export const metadata: Metadata = {
  title: "Install Corder",  // see §2 — drop "| Corder", template adds it
  alternates: { canonical: "/install/" },  // exact route, including trailing slash
  // ...rest unchanged
}
```

Same template for every other route. Path matches the route segment.

### Language declaration — fine
`<html lang="en">` set from `copy.meta.lang`. Single locale, no `hreflang` needed yet. If you ever build localised pages, add `alternates.languages` per page.

### Viewport, charset — fine
Standard Next.js defaults.

## 2. On-page: title, meta, OG, headings

### Title — DUPLICATE BRAND SUFFIX

Homepage: `<title>Corder: Record what was said</title>` — fine, no template applied because root `page.tsx` sets `title: copy.meta.title` as a string default.

Every subpage: titles like `<title>Install Corder | Corder | Corder</title>`, `<title>Privacy Policy | Corder | Corder</title>`, `<title>You're in | Corder | Corder</title>`, `<title>Not found | Corder | Corder</title>`.

Cause: each page sets `title: "Install Corder | Corder"`. Layout template is `"%s | Corder"`. Result is "Install Corder | Corder | Corder".

**Fix recipe (pick one):**

Option A (preferred — leaves template intact): edit every child `page.tsx`, drop the trailing `" | Corder"` from the page-level title.
```ts
// src/app/install/page.tsx
title: "Install Corder",   // was: "Install Corder | Corder"
```

Option B (faster, single file): change `src/app/layout.tsx` template from `"%s | Corder"` to `"%s"` and leave child titles as-is. Less ideal because future child pages might forget the suffix.

Recommendation: Option A. ~30 seconds per page, 11 pages.

Free to download for macOS 14+". Mentions product, tech, key differentiator, OS gate. Within Google's 155-160 char snippet target.

## 5. Performance and Core Web Vitals

Google PageSpeed Insights API rate-limited this run (429 across 6 attempts). Maker should run manually at https://pagespeed.web.dev/analysis?url=https%3A%2F%2Fgetcorder.com%2F&form_factor=mobile and capture LCP, INP, CLS, perf/SEO/A11y/Best-Practices scores.

### What we know from the source code

Performance budget per `projects/corder-landing/CLAUDE.md`: LCP < 1.5s on 4G, INP < 100ms, CLS = 0, total JS ≤ 80KB gzipped. Reasonable targets.

Inspected the head:
- Fonts preloaded (6 IBM Plex weights, `display: swap`). Good.
- Preconnect to Clarity, Paddle (3 hosts), Twitter pixel (when configured), Plausible. Good.
- Inline blocking scripts: motion bootstrap (tiny), Clarity loader (small but it's a network call right after parse), Twitter pixel (conditional), Paddle.js v2 from CDN (NOT async, NOT defer), Paddle init script (sync after CDN load).

### Likely Core Web Vitals issues based on source

1. **Paddle.js loaded synchronously, blocking parse.** `<script src="https://cdn.paddle.com/paddle/v2/paddle.js" />` — no `async`, no `defer`. This is the #1 contributor to TBT and likely a 200-400ms penalty on mobile. Fix: `defer` it, and move `PADDLE_INIT_SCRIPT` to fire on first interaction with a Pricing CTA (lazy-load checkout SDK on demand). The page doesn't need Paddle before the user reaches Pricing.

2. **Clarity loader inline in HEAD.** Fires before LCP. Worth either `defer`-ing the loader or wrapping it in `requestIdleCallback`. Saves ~50-150ms on mobile.

3. **CLS source: no font-display=swap fallback for serif headings.** Although `display: swap` is set, the IBM Plex Serif H1 has no `size-adjust` fallback or local fallback font matched to its metrics. On slow 3G the FOUT can shift the hero. Cheap fix: add `next/font` `adjustFontFallback` (Next 15 does this automatically for system pairs — verify). Otherwise inject a local fallback font with matching cap-height via CSS `size-adjust`.

4. **The 36 inline `<img>` tags from WorksWith** (6 logos × 3 rows × 2 viewports, or similar duplication for marquee). All `loading="lazy"`. Good. But six unique SVGs duplicated 6+ times each = ~36 image elements rendered. Switch to a single SVG sprite + `<use>` to drop DOM size by ~30 nodes.

### Items to capture once PSI returns 200

- `performance` score (target ≥ 85 mobile, ≥ 95 desktop)
- `seo` score (target 100 — only thing blocking is per-page canonical, see §1)
- `accessibility` score (target ≥ 95)
- `best-practices` score (target ≥ 95)
- LCP element ID + size + URL
- TBT breakdown by long-task source
